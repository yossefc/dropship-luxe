Contexte:
Je travaille sur un projet Node.js / TypeScript avec Prisma, Express et une integration AliExpress Open Platform.

Objectif:
Nous essayons de finaliser une integration AliExpress complete pour une application de dropshipping.

Ce qui a deja ete fait:
- migration Prisma appliquee pour stocker les credentials AliExpress
- flux OAuth AliExpress mis en place
- callback OAuth fonctionnel
- token d'acces et refresh token obtenus avec succes
- tokens stockes de maniere chiffree en base de donnees
- endpoint de statut OAuth fonctionnel
- verification confirmee que le compte AliExpress est bien autorise

Etat confirme:
- OAuth fonctionne
- le token est bien stocke
- le backend peut lire le token OAuth stocke

Probleme restant:
L'ancienne couche metier AliExpress du projet utilise des appels API historiques qui ne semblent pas compatibles avec l'application AliExpress actuelle.

Ce que nous essayons de faire maintenant:
- brancher les appels AliExpress metier sur le token OAuth stocke en base
- remplacer progressivement les anciennes methodes API par des endpoints REST officiels compatibles avec l'application actuelle
- verifier qu'un appel API metier officiel fonctionne vraiment depuis le backend

Tests deja tentes:
- appel OAuth: reussi
- test de lecture du statut OAuth: reussi
- test d'appel API metier via endpoint backend: echoue avec erreur `Invalid app Key`

Interpretation actuelle:
- le probleme n'est plus l'OAuth
- le probleme est probablement lie aux permissions/provisioning de l'application AliExpress sur les APIs metier, ou a l'incompatibilite entre l'ancien adapter et la famille d'API actuellement autorisee

Ce que je veux obtenir:
1. Identifier quelle API officielle AliExpress est reellement compatible avec cette application
2. Confirmer quels endpoints necessitent quelles permissions
3. Migrer l'adapter AliExpress existant vers les bons endpoints REST
4. Valider un premier appel metier non destructif qui reussit
5. Ensuite seulement brancher les operations plus sensibles comme commandes et tracking

Si tu m'aides, je veux une reponse tres concrete:
- diagnostic technique
- endpoints exacts a utiliser
- prerequis de permissions
- plan de migration par etapes
- exemples de requetes si necessaire
